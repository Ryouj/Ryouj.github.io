Cursor Skills 使用教程

什么是 Cursor Skills

Cursor Skills 是 Cursor AI 编辑器中的一项强大功能，允许你创建可重用的技能文件，用于指导 AI 助手执行特定任务。Skills 可以理解为”技能模板”，让 AI 助手学会如何按照你的标准和偏好完成特定工作。

Skills 与 Rules 的区别

  • Rules（规则）：存储在 .cursor/rules/ 目录，用于定义代码规范、编码标准等持久化的项目规则
  • Skills（技能）：存储在 ~/.cursor/skills/ 或 .cursor/skills/ 目录，用于指导 AI 完成特定任务或工作流程

简单来说，Rules 是”规范”，Skills 是”能力”。

Skills 的存储位置

Skills 有三种存储位置，作用范围不同：

1. 个人 Skills（全局可用）

路径：~/.cursor/skills/

  • 适用于所有项目
  • 适合个人工作习惯和偏好
  • 例如：个人代码审查标准、常用工具配置等

2. 项目 Skills（项目共享）

路径：.cursor/skills/

  • 仅适用于当前项目
  • 可以提交到 Git，团队成员共享
  • 例如：项目特定的部署流程、测试规范等

3. 系统 Skills（只读）

路径：~/.cursor/skills-cursor/

  • Cursor 内置的系统技能
  • 由 Cursor 自动管理，不要修改
  • 例如：create-skill、create-rule 等内置技能

YAML Frontmatter 要求

必需字段：

  • name：技能名称（最多 64 个字符，只能使用小写字母、数字和连字符）
  • description：技能描述（最多 1024 个字符，非空）

description 编写技巧：

  1. 使用第三人称：描述应该是客观的，不是第一人称

    • ✅ 正确："处理 Excel 文件并生成报告"
    • ❌ 错误："我可以帮你处理 Excel 文件"

  2. 包含触发词：在描述中包含关键词，帮助 AI 识别何时使用

    • ✅ 正确："从 PDF 文件提取文本和表格，填写表单，合并文档。当处理 PDF 文件或用户提到 PDF、表单、文档提取时使用。"
    • ❌ 错误："帮助处理文档"

  3. 说明 WHAT 和 WHEN：

    • WHAT：技能能做什么（具体能力）
    • WHEN：什么时候使用（触发场景）

Skills 套件

什么是 Skills 套件

Skills 套件是一组相互关联的 Skills，它们共同解决一个领域或工作流中的多个相关任务。通过组织相关的 Skills 为套件，可以：

  • 提供完整的工作流支持
  • 确保相关任务的一致性
  • 便于管理和维护
  • 提高 AI 助手的整体能力

套件结构

1. blog-post-formatting

功能：检查和格式化 Hexo 博客文章

主要能力：

  • 验证 frontmatter 格式
  • 检查中文标点符号使用
  • 验证代码块语言标识符
  • 检查标题层级

触发场景：编辑博客文章、检查文章格式、验证 frontmatter

2. blog-frontmatter

功能：为博客文章生成和优化 frontmatter

主要能力：

  • 自动生成 frontmatter 元数据
  • 根据内容提取 tags 和 categories
  • 生成符合规范的日期格式
  • 验证 frontmatter 完整性

触发场景：创建新文章、生成 frontmatter、更新文章元数据

3. blog-image-handling

功能：处理博客文章中的图片

主要能力：

  • 规范图片路径格式
  • 生成有意义的 alt 文本
  • 提供图片优化建议
  • 检查图片引用格式

触发场景：添加图片到文章、处理图片路径、优化图片

套件协同工作

这三个 Skills 可以协同工作，完成一篇博客文章的完整处理流程：

  1. 创建文章：blog-frontmatter 生成 frontmatter
  2. 添加内容：blog-image-handling 处理图片引用
  3. 格式化检查：blog-post-formatting 验证整体格式

套件设计原则

1. 单一职责

每个 Skill 应该专注于一个明确的任务，避免功能重叠。

2. 相互补充

套件中的 Skills 应该相互补充，共同覆盖完整的工作流。

3. 独立可用

每个 Skill 应该可以独立使用，不强制依赖其他 Skills。

4. 命名一致

使用一致的命名约定，便于识别套件关系：

  • 前缀相同：blog-*、code-*、test-*
  • 功能明确：名称清楚说明功能

创建套件的最佳实践

  1. 识别工作流：分析你的工作流程，识别可以技能化的步骤
  2. 拆分任务：将复杂工作流拆分为独立的、可重用的任务
  3. 设计接口：确保 Skills 之间的输入输出格式一致
  4. 文档说明：在套件的 README 中说明套件的整体用途和 Skills 之间的关系
  5. 迭代优化：根据使用反馈不断优化套件

如何使用 Skills

自动触发机制

AI 助手会根据以下信息自动选择合适的 Skills：

  1. 用户请求内容：分析用户的问题和需求
  2. Skill 描述：匹配 description 中的关键词
  3. 上下文信息：当前打开的文件、项目类型等

例如，当你问”帮我审查这段代码”时，AI 会自动查找并应用代码审查相关的 Skills。

手动使用方式

虽然 Skills 主要是自动触发的，但你可以：

  1. 明确提及：在对话中明确提到技能名称

  2. 查看可用 Skills：在 Cursor 中查看已安装的 Skills

    • 个人 Skills：~/.cursor/skills/
    • 项目 Skills：.cursor/skills/

  3. 创建自定义 Skills：根据你的需求创建新的 Skills

验证 Skills 是否生效

创建 Skill 后，可以通过以下方式验证：

  1. 重启 Cursor：确保新创建的 Skills 被加载
  2. 测试触发：使用描述中的关键词测试是否能触发
  3. 检查行为：观察 AI 是否按照 Skill 中的指导执行

最佳实践总结

创建 Skills 时

  • ✅ 保持 SKILL.md 文件在 500 行以内
  • ✅ 使用第三人称编写 description
  • ✅ 包含具体的触发关键词
  • ✅ 提供实际可用的示例
  • ✅ 使用一致的术语
  • ✅ 避免时间敏感的信息

避免的反模式

  • ❌ 使用 Windows 风格的路径（使用 / 而不是 \）
  • ❌ 提供太多选项（选择一个默认方案）
  • ❌ 使用模糊的技能名称（如 helper、utils）
  • ❌ 在描述中使用第一人称

总结

Cursor Skills 是一个强大的功能，可以让你定制 AI 助手的行为，使其更符合你的工作习惯和项目需求。通过创建和使用 Skills，你可以：

  • 标准化工作流程
  • 提高 AI 助手的准确性
  • 在团队中共享最佳实践
  • 减少重复性指导
